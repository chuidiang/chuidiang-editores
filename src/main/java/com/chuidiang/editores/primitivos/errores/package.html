En este paquete están las clases que producen algún tipo de aviso cuando el usuario introduce en un editor un valor incorrecto.

Todos las clases de tratamiento de error heredan de InterfaceTratamientoError, que tiene dos métodos.

    • correcto( Editor<Tipo> editor ): en el que se le indica que el contenido del editor es correcto y debe eliminar la marca o aviso que le haya puesto
      tomaError(String error, InterfaceEdicionDatos<Tipo> editor ) donde se le indica que el editor tiene un valor incorrecto y se le pasa un String con el motivo por el que no es correcto. La clase debe marcar/avisar de que dicho editor no tiene un valor correcto

    En este paquete hay varias clases que implementan estan interface:

    • ErrorConAviso: Muestra una ventana de aviso con el error.
      ErrorConColor: Pone el background del editor de un color para indicar el fallo, por ejemplo rojo.

  • Adicionalmente hay una clase Verificador, que pasándola a cualquier Editor impide que pueda perder el foco hasta que el valor del editor sea correcto
